# Subsystem Overview

## Purpose
The `code/macosx` subsystem implements the macOS platform layer for Quake III Arena, bridging the engine's renderer, input, and display management to Apple's AppKit, CoreGraphics, and OpenGL frameworks. It handles macOS-specific concerns including OpenGL context management, raw mouse input via private CGS APIs, hardware display mode and gamma control, and the application lifecycle entry point.

## Key Files

| File | Role |
|---|---|
| `Q3Controller.h` / `.m` | AppKit `NSObject` subclass; macOS application controller and engine entry point (`quakeMain`) |
| `macosx_local.h` | Shared platform header; declares `glwstate_t`, the global `glw_state`, and `OSX_*` GL context macros |
| `macosx_glimp.h` / `.m` | OpenGL framework includes; optional `CGLMacro` context optimization path; GL visibility/pause functions |
| `macosx_display.h` / `.m` | Display mode query, gamma ramp management, and display fade/unfade during mode switches |
| `macosx_input.m` | Raw mouse and keyboard input layer; consumes `CGMouseDeltaFix` and `CGPrivateAPI` |
| `macosx_qgl.h` | Autogenerated `qgl*` wrapper layer over OpenGL 1.x; optional call logging and GL error checking |
| `CGMouseDeltaFix.h` / `.m` | Shim working around `CGGetLastMouseDelta` platform inconsistencies; provides stable mouse delta queries |
| `CGPrivateAPI.h` | Mirrors Apple's private CGS event API to receive global mouse-moved events outside window focus |
| `macosx_timers.h` | Optional OmniTimer profiling instrumentation; collapses to no-ops when `OMNI_TIMER` is undefined |

## Core Responsibilities

- Provide the macOS application lifecycle entry point via `Q3Controller` and its `quakeMain` method, wired through a `.nib` file
- Manage the OpenGL rendering context (`CGLContextObj`) stored in `glwstate_t` / `glw_state`, with safe get/set/clear via `OSX_*` macros
- Wrap all OpenGL 1.x calls in `qgl*` shims that optionally log calls and check for GL errors after each invocation
- Query and set hardware display modes and gamma ramp tables, including fade/unfade transitions during fullscreen mode switches
- Deliver raw mouse delta input by working around `CGGetLastMouseDelta` instability via `CGMouseDeltaFix`, and by hooking undocumented `CGSRegisterNotifyProc` to receive global mouse-moved events
- Optionally instrument renderer and collision code paths with hierarchical OmniTimer profiling nodes

## Key Interfaces & Data Flow

**Exposes to other subsystems:**
- `glwstate_t` / `glw_state` — central OpenGL window state consumed by the renderer and input layers across the macOS platform files
- `OSX_GetCurrentContext()` / `OSX_SetCurrentContext()` / `OSX_ClearCurrentContext()` — safe GL context access macros used by `macosx_glimp.m` and the renderer
- `Sys_GetMatchingDisplayMode()` — display mode query consumed by the renderer during `GLimp_Init`
- `qgl*` wrappers (via `macosx_qgl.h`) — all GL calls from the renderer flow through these shims
- `InitializeTimers()` — optional profiling init consumed during engine startup

**Consumes from other subsystems:**
- `qcommon.h` — engine core types (`qboolean`, `sysEventType_t`, `fileHandle_t`)
- `tr_local.h` — renderer types (`glconfig_t`, `qboolean`) required by display management
- `ApplicationServices.framework` — `CGDirectDisplayID`, `CGMouseDelta`, CoreGraphics display and event APIs
- `AppKit.framework` — `NSObject`, `NSPanel`, `NSWindow`, `NSEvent`, `NSOpenGLContext`
- `OpenGL.framework` — `gl*`, `glu*`, CGL types, and optionally `CGLMacro.h`
- `OmniTimer.framework` — `OTStackNode`, `OTStackPush`/`OTStackPop` (optional; open-source release omits this)
- Private CGS framework — `CGSRegisterNotifyProc` resolved at runtime for global mouse event notifications

## Runtime Role

- **Init:** `Q3Controller` receives the AppKit application launch notification and calls `quakeMain` to transfer control to the engine. Display modes are queried and a matching `CGDirectDisplayID` is selected. The GL context is created and stored in `glw_state`. `InitializeTimers()` is called if `OMNI_TIMER` is defined. `CGMouseDeltaFix` is initialized to patch the mouse delta query path.
- **Frame:** Each frame the input layer calls the `CGMouseDeltaFix` shim to read raw mouse deltas, and the `CGSRegisterNotifyProc` notification path delivers global mouse-moved events. The renderer issues all GL calls through `qgl*` wrappers, with optional per-call logging and error checking. OmniTimer macros (`OTSTART`/`OTSTOP`) bracket renderer and collision work if profiling is active.
- **Shutdown:** Display fade functions are called during fullscreen teardown. Gamma tables are restored via the stored `glwgamma_t` data. The GL context is cleared via `OSX_ClearCurrentContext()` and the display is released.

## Notable Implementation Details

- `CGPrivateAPI.h` uses Apple's **undocumented, private** CGS notification API (`CGSRegisterNotifyProc`) to receive mouse-moved events globally, not just when the window has focus. This registration is resolved at runtime rather than linked at compile time.
- `macosx_glimp.h` supports a `USE_CGLMACROS` build path that replaces every CGL call with a macro-rewritten version embedding the context pointer inline, eliminating a per-call context lookup at the cost of coupling all including translation units to `glw_state`.
- `macosx_qgl.h` is explicitly autogenerated (noted in file purpose) and poisons bare `gl*` symbols via `#define` redirects, enforcing that the renderer never calls OpenGL directly — a pattern shared with the other id Tech 3 platform backends.
- The `DEDICATED` preprocessor guard in `Q3Controller.h` strips all AppKit UI code (splash panel outlet, paste/quit IBActions) from dedicated server builds.
- `macosx_timers.h` declares `OTStackNode*` globals for named profiling points across renderer and collision paths, but the `OmniTimer` framework is not present in the open-source release; all instrumentation compiles away to no-ops.
