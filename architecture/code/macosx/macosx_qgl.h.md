# code/macosx/macosx_qgl.h

## File Purpose
Autogenerated macOS-specific header that wraps every standard OpenGL 1.x function in a `qgl`-prefixed inline shim. Each shim optionally logs the call to a debug file and/or checks for GL errors after the call, then forwards to the real `gl*` function. A block of `#define` macros at the end redirects all bare `gl*` names to the error-message symbols, forcing callers to use the `qgl*` versions.

## Core Responsibilities
- Provide `qgl*` inline wrappers for every OpenGL 1.x core function (~200+ functions)
- Conditionally log GL call parameters to a debug file when `QGL_LOG_GL_CALLS` is defined
- Conditionally call `QGLCheckError()` after each GL call when `QGL_CHECK_GL_ERRORS` is defined
- Track nested `glBegin`/`glEnd` depth via `QGLBeginStarted` to suppress error checks inside a primitive block
- Poison all bare `gl*` symbols via `#define gl* CALL_THE_QGL_VERSION_OF_gl*` to enforce use of wrappers
- Provide `_glGetError()` as an unguarded bypass to avoid infinite recursion inside `QGLCheckError`

## Key Types / Data Structures
None.

## Global / File-Static State
| Name | Type | Scope | Purpose |
|---|---|---|---|
| `QGLLogGLCalls` | `unsigned int` (extern) | global | Non-zero to enable per-call logging |
| `QGLBeginStarted` | `unsigned int` (extern) | global | Nesting counter for `glBegin`/`glEnd`; suppresses error checks inside primitive blocks |

## Key Functions / Methods

### _glGetError
- Signature: `static inline GLenum _glGetError(void)`
- Purpose: Calls the raw `glGetError()` without going through the wrapper, preventing infinite recursion when `QGLCheckError` itself needs to query errors.
- Inputs: None
- Outputs/Return: `GLenum` error code
- Side effects: None
- Calls: `glGetError` (direct)
- Notes: Must bypass the `#define glGetError` poison; declared before the poisoning block.

### qglBegin
- Signature: `static inline void qglBegin(GLenum mode)`
- Purpose: Wraps `glBegin`; increments `QGLBeginStarted` so downstream wrappers skip error checks while inside a primitive.
- Inputs: `mode` — GL primitive type
- Outputs/Return: void
- Side effects: Increments `QGLBeginStarted`; may write to debug log
- Calls: `glBegin`, `QGLCheckError` (conditionally), `fprintf`/`QGLDebugFile` (conditionally)

### qglEnd
- Signature: `static inline void qglEnd(void)`
- Purpose: Wraps `glEnd`; decrements `QGLBeginStarted` and triggers error check when the counter reaches zero.
- Inputs: None
- Outputs/Return: void
- Side effects: Decrements `QGLBeginStarted`; may call `QGLCheckError`
- Calls: `glEnd`, `QGLCheckError` (conditionally)

### Notes
- All other ~200+ `qgl*` functions follow an identical pattern: optional `fprintf` log, forward to `gl*`, optional `QGLCheckError` guarded by `!QGLBeginStarted`. They are not individually documented here as the pattern is fully uniform.

## Control Flow Notes
This header is included by macOS renderer/glimp code during init and every render frame. It has no init or shutdown logic of its own. The wrappers are active for every OpenGL call made by the macOS renderer path at runtime. The `QGLBeginStarted` counter ensures error checking is deferred until after `glEnd` completes a primitive.

## External Dependencies
- Standard OpenGL headers (implicit via including code): all `gl*` functions, GL types
- `QGLCheckError` — defined in a companion `.m`/`.c` file (`macosx_qgl.m` or similar)
- `QGLDebugFile()` — returns a `FILE*` for debug output; defined elsewhere
- `QGLLogGLCalls`, `QGLBeginStarted` — extern globals, defined in companion source
