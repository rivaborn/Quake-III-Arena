# q3radiant/ChildFrm.h — Enhanced Analysis

## Architectural Role

`CChildFrame` is an MFC (Microsoft Foundation Classes) UI component in the **Q3Radiant level editor**, an offline tool separate from the runtime engine. It manages individual document windows in a Multiple Document Interface (MDI) application, allowing mappers to work with multiple BSP maps simultaneously. This class is purely Windows-specific UI infrastructure with no data flow into runtime game logic.

## Design Patterns & Rationale

**MDI (Multiple Document Interface) pattern**: Standard Windows UI pattern where a parent frame contains multiple child windows, each representing a document. This was idiomatic in the late 1990s–early 2000s Windows desktop applications.

**MFC classWizard-generated structure**: The `//{{AFX_*` comment markers indicate this skeleton was generated by Visual Studio's ClassWizard tool. The file follows strict MFC conventions (see the preserved block comments in lines 65–69).

**Minimal override strategy**: Only `PreCreateWindow` is overridden, suggesting the default MDI child behavior is mostly acceptable. The implementation (in the missing `.cpp` file) likely modifies window style flags or initial dimensions.

## Learning Notes

1. **Windows-only legacy**: Pure MFC code reflects that Q3Radiant (shipped ~2001) was Windows-only. Modern map editors would use cross-platform toolkits (Qt, wxWidgets, or custom OpenGL frameworks).

2. **Message-map idiomatic pattern**: `DECLARE_MESSAGE_MAP()` / `DECLARE_DYNCREATE()` are quintessentially MFC—compile-time binding of window messages to handler functions. Contrast with modern event systems (Qt signals/slots, C# events).

3. **Separation of concerns**: The tool's UI layer (this class) is completely decoupled from runtime engine logic. Q3Radiant is a **compile-time tool** (like q3map, bspc, q3asm), not a runtime component.

4. **MDI as architectural metaphor**: Each `CChildFrame` implicitly represents one loaded map file, allowing side-by-side editing or batch processing scenarios.
